# PI Firm Offer Creator

Create high-quality, realistic, and market-ready offer concepts for Personal Injury law firms.

## Purpose

Use this skill to produce complete offer packages that:
- Fit Blue Tusk's real capabilities and pricing context
- Follow the Offer Creation Workflow end-to-end
- Tie each offer to PI law firm value drivers
- Include practical marketing examples across multiple channels

## Required references

Read these files before drafting offers:
- `references/blue-tusk-capabilities.md`
- `references/pi-law-firm-value-drivers.md`
- `references/agp-offer-creation-workflow.md`
- `references/agp-offer-creation-workflow-EXAMPLE.md`
- `references/hormozi-attraction-offers.md`

## Core constraints

1. Keep offers realistic for Blue Tusk to deliver now (do not invent unsupported services).
2. Ground claims in plausible assumptions and simple math.
3. If a guarantee is included, state any qualifying conditions clearly.
4. Keep language specific to PI firms, not generic SMB language.
5. Every offer must map to one or more core value drivers.

## Mandatory output components

For each offer idea, ALWAYS include all of the following:

1. Offer summary
- Offer name
- One-sentence positioning
- Ideal PI firm profile
- Delivery model (advisory, implementation, hybrid)

2. Completed Offer Creation Workflow
- Fill every section from the AGP workflow template:
  - Step 1: Niche Definition
  - Step 2: Outcome Definition
  - Step 3: Time Frame Definition
  - Step 4: Mechanism Definition
  - Step 5: Risk Reversal
  - Step 6: Polarisation
  - Step 7: Pricing
- Do not leave placeholders or blank bullets.

3. Value driver alignment
- Explicitly map the offer to the PI value drivers from the reference file.
- Show the chain of impact using this pattern:
  - We improve X -> which creates Y -> enabling Z -> resulting in business outcome Q.

4. Marketing examples
- Instagram video post examples:
  - At least 3 short-form concepts
  - Each includes hook, talking points, CTA
- Website landing page examples:
  - At least 1 full landing page outline with section headers and copy direction
  - Include headline options, proof elements, offer details, CTA
- Cold email script examples:
  - At least 2 outbound sequences (minimum 3 emails each)
  - Include subject lines, body copy, and clear CTA per email

## Workflow

1. Determine offer count requested by the user. If unspecified, default to 3 distinct offers.
2. Pull constraints and capabilities from Blue Tusk context.
3. Use the AGP workflow to draft each offer end-to-end.
4. Validate each offer against PI value drivers and add explicit mapping.
5. Build channel-specific marketing examples for each offer.
6. Run a final realism check:
- Is this deliverable by Blue Tusk now?
- Are pricing and outcomes coherent?
- Are marketing claims supportable and compliant in tone?

## Quality checklist

Before finalizing, verify all are true:
- No workflow fields are blank.
- Offer mechanisms are operationally clear.
- Pricing band aligns with documented Blue Tusk positioning.
- Value-driver mapping is explicit and causal.
- Instagram, landing page, and cold email examples are present for every offer.
- Messaging emphasizes ROI, modernization, and measurable outcomes.
