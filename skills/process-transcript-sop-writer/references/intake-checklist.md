# SOP Intake Checklist (Process Transcript to SOP)

Use this checklist before finalizing an SOP.

## A) Process basics
- Process name
- Process goal/outcome
- Intended audience (who will follow the SOP)
- Primary owner (person or role)

## B) Trigger and entry point
- What starts the process?
- Where does the initial input come from?
- How often does the process run?

## C) Tools, programs, and platforms
- Full list of tools used
- What each tool does in this process
- Required access/permissions for each tool

## D) Files and folders
- File names used or created
- Folder locations (exact paths if available)
- Naming conventions
- Storage system (Google Drive, local folder, shared drive, etc.)

## E) Step-by-step flow
- Ordered steps from start to finish
- Responsible person/role per step
- Input and output of each step
- Handoffs between tools or people

## F) Validation and quality checks
- How to verify each major step succeeded
- Final success criteria
- Approval checkpoints (if any)

## G) Exceptions and troubleshooting
- Common failure points
- What to do when a step fails
- Who to notify/escalate

## H) Critical dependencies and change warnings
- Integrations, IDs, mappings, folder structures, formulas, webhooks, credentials, or schedules that the process depends on
- Why each dependency is fragile or important
- Safe-change procedure (test first, owner approval, rollback plan)

## I) Missing information rule

If any section above is missing critical information, ask targeted clarifying questions and delay final SOP finalization until enough context is available.
