# Process Transcript SOP Writer

## Problem
AI and automation agencies often have process knowledge trapped in transcripts, voice notes, or informal walkthroughs instead of standardized documentation.

## Pain Point
Without a clear SOP, non-technical team members cannot reliably run or maintain processes, and key implementation details (tools, files, folders, dependencies) are missed until something breaks.

## Description
Create a skill that takes a transcript or rough process description and turns it into a complete, beginner-friendly SOP that explains how tools, files, and folders work together. The skill must ask for missing context before finalizing and include a dedicated section highlighting critical dependencies that should not be changed without care.

## Value Driver
Standardized SOP generation reduces onboarding friction, improves consistency, lowers support burden, and prevents avoidable automation failures caused by undocumented or fragile dependencies.

## Optional Notes
- Users: AI and automation agency owners, operations managers, implementation specialists, non-technical assistants
- Current workaround: Manual SOP writing from scattered notes and repeated follow-up messages
- Success signal: Team members with low technical knowledge can execute the process correctly from the SOP alone
- Scope boundaries: Focus on process documentation quality and completeness, not building the automation itself
