# AI & Automation Agency Client Onboarding Skill

## Problem
After clients share ideas and proposed solutions, onboarding often stalls because required access details, business context, and implementation inputs are missing or fragmented.

## Pain Point
Agency teams waste early project time chasing logins, permissions, goals, and process details across multiple back-and-forth messages, delaying execution and reducing client confidence.

## Description
Create a skill that converts a client’s idea and proposed solution into a structured onboarding intake. It should request all required implementation inputs up front (for example tool logins, account access paths, business context, current workflows, constraints, goals, and approval contacts), then return a clear onboarding packet and checklist so the client can step back while the agency executes.

## Value Driver
A consistent onboarding process reduces kickoff delays, prevents critical missing information, and helps the agency move from sales-to-delivery faster with fewer client interruptions.

## Optional Notes
- Users: AI and automation agency owners, project managers, implementation specialists, new clients
- Current workaround: Ad hoc kickoff calls and scattered requests via chat/email
- Success signal: Clients provide complete onboarding details within the first intake cycle and project build can start immediately
- Scope boundaries: Focus on onboarding data collection and handoff readiness, not the full downstream automation build
