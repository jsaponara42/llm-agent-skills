# Skill Idea Process

This repo uses a simple, fast process for skill ideas.

Goal: make idea capture non-daunting so you can log ideas quickly and improve later.

## 2-minute capture

Create one markdown file in `skill-ideas/ideas/` with four fields:
- Problem
- Pain Point
- Description
- Value Driver

Do not overthink it. Short and rough is fine.

## 10-minute refine (optional)

If you want to improve an idea, do one short refinement pass:
- Clarify who the user is
- Add one concrete use case
- Add one success signal
- Add one boundary (what it should not do)

## Naming convention

Use:

`YYYY-MM-DD-short-kebab-name.md`

Examples:
- `2026-03-13-support-ticket-summarizer.md`
- `2026-03-13-proposal-draft-helper.md`

## Where everything lives

- Idea files: `skill-ideas/ideas/`
- Idea template: `skill-ideas/IDEA_TEMPLATE.md`
- Master index: `skill-ideas/SKILL_IDEAS.md`
- Idea writing skill: `skills/skill-idea-describer/SKILL.md`

## Definition of done

An idea is ready for next-step scoping when:
- The problem is specific
- The pain point is real and repeatable
- The description says what the skill does
- The value driver is clear
